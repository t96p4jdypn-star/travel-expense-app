# 外部連携なし・ブラウザ内運賃台帳仕様

## 目的

有料サービス、外部アプリ、外部経路APIを使わず、ブラウザ内だけで旅費の自動補完・再計算・登録・一覧確認・出力を完結させる。

## 調査結果と採用理由

- GTFSにはゾーン別・距離別・時間帯別などの運賃を表現できる公式仕様がある。
  - https://gtfs.org/ja/getting-started/features/fares/
- GTFS-JPは駅・時刻・運賃等を扱えるが、全国すべての鉄道会社の最新IC運賃を統一条件で取得できる単一の無償配布データではない。
  - https://www.mlit.go.jp/commmmons/document/007/
- 公共交通オープンデータセンターはユーザー登録無料だが、API登録と外部連携が必要で、データ提供範囲も事業者ごとに異なる。
  - https://www.odpt.org/
- 鉄道会社サイトの画面取得や無許可スクレイピングは、CORS、画面変更、利用規約、再現性の問題があり採用しない。
- 以上から、利用者が業務で確認した区間とIC料金を端末内に一度登録し、次回から同一区間を自動計算する方式を採用する。

## 変更内容

### 1. ブラウザ内運賃台帳

- 出発駅、到着駅、有料区間、IC料金、経路、登録日時、最終利用日時、利用回数をIndexedDBへ保存する。
- 同じ出発駅・到着駅は逆方向でも同額候補として利用できる。ただし有料区間の表示方向は入力方向へ合わせる。
- 運賃台帳はJSONバックアップ・復元へ含める。
- 登録内容は利用者が編集・削除できる。

### 2. 入力から確定まで

1. 日付、時刻、行き先を入力する。
2. 訪問先、同日の直前行、曜日別出発地、運賃台帳から駅・理由・区間・IC料金を自動補完する。
3. 利用者が出発駅、到着駅、有料区間、IC料金、理由を必要に応じて修正する。
4. 「再計算」で定期券判定、申請額、0円、重複警告を更新する。
5. 「確定して登録」で旅費行を確認済みにし、訪問先・履歴・運賃台帳へ保存する。

### 3. 再計算

- 登録済み区間が見つかった場合は登録IC料金を適用する。
- 見つからない場合は入力中のIC料金を維持し、未登録として表示する。
- 全区間が有効な定期券内なら申請額を0円にする。
- 運賃台帳の金額は「確定して登録」を押すまで更新しない。
- 再計算後も行は未確認のままとし、自動確定しない。

### 4. 登録状況

- 主メニューに「登録状況」を追加する。
- 対象月の全旅費行を状態別に一覧表示する。
- 確認済み件数、未確認件数、保留件数、0円件数、出力対象件数、申請合計を表示する。
- 運賃台帳を区間、IC料金、登録日、利用回数とともに一覧表示する。
- 一覧から「コピー出力」「Excel出力」へ移動できる。

## 変更しない範囲

- OCRは試験機能として残す。
- スクリーンショットの貼り付け保存を残す。
- 月管理、1日経路、定期券、重複警告、状態管理、コピー出力、Excel出力を維持する。
- 外部AIへ画像・予定情報を送らない。

## 完了条件

- ネットワークを使わずに登録済み区間を自動計算できる。
- 未登録区間を修正し、再計算・確定登録できる。
- 確定した区間が次回入力で自動利用される。
- 登録状況一覧から出力画面へ進める。
- 既存データを失わず移行できる。
- 自動テスト、型チェック、ビルド、GitHub push、公開確認が完了している。
