# 行き先中心入力・スクリーンショット保存仕様

## 目的

予定OCRの精度に業務を依存せず、利用者が「日付と行き先」を入力するだけで旅費行を作れるようにする。繰り返し使うほど、最寄駅、出発駅、経路、移動理由の入力を減らす。

## 変更内容

### 1. 入力の主経路

- 月間画面の先頭に「かんたん追加」を設置する。
- 必須入力は日付と行き先。既知の行き先選択時は最寄駅と移動理由を自動補完する。
- 同日の既存行がある場合は、直前行の到着駅を次の出発駅にする。
- 同日の最初の行は曜日別標準出発地、自宅、出勤先の順で解決する。
- よく使う行き先、最近使った行き先をワンタップ候補にする。
- 初回の行き先だけ最寄駅と移動理由を入力し、次回以降は訪問先候補として再利用する。
- 作成した行は未確認とし、利用者が経路・金額を確認してから確認済みにする。

### 2. IC運賃の自動計算

- 第一候補は駅すぱあと API の経路探索とする。
- ブラウザへアクセスキーを配布せず、公開Webアプリのサーバー側からAPIを呼び出す。
- 外部へ送る情報は出発駅、到着駅、利用日・時刻に限定する。スクリーンショット、予定名、業務内容、氏名は送らない。
- ICカード運賃を優先し、経路、運賃、運賃改定対応状態、取得日時を旅費行へ記録する。
- 定期券区間は登録済み情報をもとに控除する。全区間が定期内なら0円として内部保持し、通常出力しない。
- API未設定・失敗時は確定履歴の同一区間から経路、理由、直近運賃を候補表示する。ただし履歴運賃は自動確定せず、「履歴・要確認」と取得日を表示する。
- 新規区間かつAPIが使えない場合は金額だけ手入力できる。以後の確定時に履歴へ保存する。

### 3. スクリーンショット保存

- 予定取込画面に貼り付け領域を設け、Mac/Windows/iPhone/iPadのクリップボード画像を保存できるようにする。
- 画像はIndexedDBに保存し、外部AIおよび経路APIへ送らない。
- 対象月ごとにサムネイル、保存日時、削除、拡大表示を提供する。
- 画像選択でも参考画像として保存できる。
- JSONバックアップへ画像を含め、復元できるようにする。

### 4. OCRの扱い

- 既存の画像/PDF OCRは削除しない。
- 「試験機能」と明記し、折りたたんだ補助領域へ移す。
- OCR結果は引き続き自動確定しない。

### 5. 入力軽減の優先順位

1. 同じ行き先・同じ区間の確定履歴
2. 登録済み訪問先
3. 最近使用した行き先
4. 利用回数の多い行き先
5. 曜日別出発地・戻り先
6. 利用者の手入力

## 変更しない範囲

- 月単位管理、複数行、1日の時刻順経路、定期券、重複警告、状態管理、IndexedDB、JSONバックアップ・復元は維持する。
- コピー出力、Excel自動作成、20行ごとの分割、申請済み操作は維持する。
- スクリーンショットや予定情報を外部AIへ送らない方針は維持する。

## 採用技術と調査根拠

- 駅すぱあと API は公式に最新の電車・バス運賃、ICカード運賃、定期券を考慮した経路操作を提供しており、交通費精算用途に適合するため第一候補とする。
  - https://ekispert.jp/products/api
  - https://docs.ekispert.com/v1/api/search/course/extreme.html
  - https://docs.ekispert.com/v1/tips/get-course-for-ic-fare/
- API利用にはアクセスキーが必要。評価環境は90日間無料で、継続利用料金は利用条件により問い合わせとなる。
- Google Routes APIも交通機関運賃を返せるが、全区間の運賃を判定できる場合に限られ、交通費精算での確実性が低いため第一候補にしない。また結果保存等の利用条件も追加検討が必要。

## 実装計画・作業項目

1. データ形式を後方互換で拡張し、スクリーンショットと運賃出所を保存する。
2. かんたん追加、候補順位、日内の出発駅推定、履歴運賃候補を実装する。
3. スクリーンショット貼り付け・一覧・削除を実装し、OCRを試験機能へ移す。
4. 駅すぱあと API 用サーバー経路と、キー未設定時の安全なフォールバックを実装する。
5. 自動テスト、PC幅・iPhone幅相当のレスポンシブ確認、ビルド、公開確認を行う。

## 完了条件

- 既知の行き先は日付と行き先だけで旅費行を作成できる。
- 同日の2件目は1件目の到着駅から自動的につながる。
- API利用可能時はIC運賃を取得し、利用不可時はデータを失わず手入力・履歴候補へ戻れる。
- スクリーンショットを貼り付け保存・表示・削除・バックアップ復元できる。
- OCR、コピー出力、Excel出力が残っている。
- テスト、GitHub push、公開URL確認が完了している。
